## Global Constraints

- **Byte-parity discipline:** every pure function is proven against committed JSON fixtures generated by `tools/port/gen_rebalance_fixtures.py` (NEW, in `/home/sat/bin/cl_revenue_ops-port` — see T1) running the real Python modules; fixtures are committed in this repo under `fixtures/rebalance/`. - **Error strings are contracts** (substring/prefix-matched by the engine's failure classifier, the audit, and the defibrillator): `local_budget_block`, `zero_budget_blocks_auto_rebalance`, `engine_busy`, `dest_inflight`, `cycle_already_running`, `route_pricing_failed: `, `native_route_invalid: `, `native_route_over_budget: `, `native_sendpay_error: `, `payment_pending_timeout: `, `no_route`, `temporary_channel_failure`, `fee_insufficient`, `incorrect_cltv_expiry`, `permanent_failure`, `governor_block:`. Centralized once in `errors.rs`; never retyped inline.
- **payment_pending is safety-critical:** when a payment is unresolved (`waitsendpay` code 200 / proxy timeout / status pending) the executor sets `payment_pending=true`, NEVER deletes invoice/payment, NEVER emits exclusions; the engine NEVER retries on top of a pending payment (break on pending, P8-001) and HOLDS the budget reservation only when `payment_pending AND payment_hash` (P4-007/P4-009); no-hash pending records `failed` and releases.
- **Budget rail invariants (MERGED unified rail, `revops_db::budget`):** pass the FULL unified budget to `reserve_budget` — never pre-subtract external costs (P4-016); reservation amount is the pair's max fee (`effective_budget_sats` when priced, else `pair_budget_sats`); budget blocks → history status `skipped` (never `failed`), never futility strikes; `reserve_budget` `Err` maps to refuse-and-continue at the cycle boundary — never `?` through the auto-cycle (3B checklist); spend-journal hooks (`note_spend_reserved/released/settled`, P4-019/P4-022) wired or documented fail-safe-over-hold.
- **Money helpers:** `base_to_sats_ceil` for fees (never undercount cost), `base_to_sats_floor` for balances; integer-exact ceil idioms `-(-a*b // c)` and `(x + 999_999) / 1_000_000` ported as checked integer math; msat fields parse both `u64` and `"Nmsat"` strings.
- **Dry-run discipline (like fees):** the engine plans and prices freely; ALL sendpay-path RPCs sit behind `PaymentMode::DryRun|Live` and all DB writes behind `RebalanceStore` (dry-run impl = JSONL journal). `PaymentMode::Live` is constructed ONLY by the cutover task (not in this plan); a source-scan test asserts no other construction site.
- **Single-flight everywhere:** cycle lock non-blocking (`run_cycle` skips `cycle_already_running`; `execute_candidate` fails `engine_busy`); in-flight-dest guard is a COUNTED map, not a set (P4-008).
- **Parallel-safe waves per the established pattern:** worktree isolation, TDD, fixture generators on a port-repo branch consolidated at wave merges (the T2/T3, T8, T10 precedent), controller commits merges. Gates per task: fmt, clippy `-D warnings`, `cargo test --workspace` dev AND release.
